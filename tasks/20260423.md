## 概要

同一性と等価性の混同。「同じ住所なのに別オブジェクト」「IDが同じなのに属性が違う」混乱を描き、
Entity と Value Object の判断基準を明確にする。

## 技術ポイント

- アンチパターン: すべてにIDを振る、eq と == の混同、可変Value Object
- 解決策: 識別子による同一性（Entity）vs 属性による等価性（Value Object）
- Moo での実装: eq 演算子オーバーロード、不変属性、ファクトリメソッド

- 種別: 続編
- 関連: Value Object（3/10）「偽造された身分証」— Value Objectの概念をEntity対比で深堀り
