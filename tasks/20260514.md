## 概要

クライアントが複数のマイクロサービスと直接通信し、結合点が爆発する事件。
API Gateway で「一つの窓口」を提供し、クライアントとバックエンドの間に防壁を築く。

## 技術ポイント

- アンチパターン: クライアント→マイクロサービス直接呼び出しの氾濫
- 解決策: Gateway集約、リクエストルーティング、レスポンス合成
- Moo での実装: Gatewayオブジェクト、ルーティングテーブル、レスポンスアグリゲーター
