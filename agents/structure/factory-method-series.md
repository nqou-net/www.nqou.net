# 連載構造案：生成を遊ぶ新シリーズ

調査結果: `content/warehouse/factory-method-series.md` の調査結果に基づく

## 前提情報

- **技術スタック**: Perl v5.36以降（signatures、postfix dereference）、Mooによるオブジェクト指向
- **想定読者**: Perl入学式卒業程度の初学者
- **想定ペルソナ**: 「Mooで覚えるオブジェクト指向プログラミング」シリーズを読了し、次の実践題材を探している読者
- **学習目標**:
  - 継承とオーバーライドの使い所を体感する
  - Factory Methodの本質を体験として理解する
  - SOLID原則のうちOCP/DRYを実例で実感する
- **位置づけ**: Moo基礎シリーズの応用編
- **ストーリー**: まず動かす → 追加機能で壁に当たる → リファクタリングで解決 → 完成コードで達成感
- **制約**:
  - 1記事1概念
  - コード例2つまで
  - 回の最後には完成コードを示す
  - シリーズ名にデザインパターン名を含めない
  - 既存シリーズと題材が重複しない

### 既存シリーズとの差別化

| シリーズ | 題材 | パターン | アプローチ |
|---------|------|---------|-----------|
| PerlとMooでレポートジェネレーターを作ってみよう | レポート生成 | Factory Method | 業務レポート生成 |
| PerlとMooでモンスター軍団を量産してみよう | モンスター量産 | Prototype | cloneによる複製 |
| Perlでローグライク通知システムを作ろう | 通知システム | Observer | 登録/通知の分離 |
| PerlとMooでWebスクレイパーを作ってみよう | Web収集 | Template Method | 共通手順の抽象化 |
| Mooで作る簡易テキストエディタ | テキスト編集 | Command | 操作オブジェクト化 |
| 本シリーズ（新規） | ドローン工房/脱出ゲーム/ドリンク合成 | Factory Method | 生成対象の切り替え体験 |

## 案A: 「ドローン工房」アプローチ

### シリーズ名案

「PerlとMooでカスタムドローン工房を作ってみよう」（全6回）

### 特徴・アプローチ

任務別に装備が変わるドローンを量産する工房を作る。救援、偵察、配送などの任務を追加するほどif/elseが増え、Factory Methodで生成責務を整理する流れを作る。

### USP（独自の価値提案）

「なぜこのシリーズにお金を払う価値があるのか？」

- ミッション追加が楽しくなる設計の気持ちよさを体験できる
- ハードウェアっぽい題材で抽象化のセンスを身に付けられる
- 生成パターンの核心を“工房運営”として語れる

### メリット

- 実務のジョブ管理やバッチ生成に置き換えやすい
- 追加任務のたびに設計が崩れる体験を作りやすい
- 完成時に成果物が視覚的に想像しやすい

### デメリット

- ドローンの機能設定が抽象的に感じる可能性がある
- 現実のドローン事情を深掘りしすぎると脱線する

## 推薦案とその理由

### 推薦：案A「ドローン工房」アプローチ

### 推薦理由

1. **ペルソナとの適合性**
   - ガジェット制作の物語が“作って動かす楽しみ”を強調できる
   - 既存シリーズとテーマが明確に分離できる

2. **検索意図との適合性**
   - ドローン/工房/自動化のキーワードが新しい検索経路になる
   - Factory Methodを学びたい読者にも、ものづくり系の読者にも届く

3. **学習効果**
   - 任務追加のたびに設計が崩れる体験を作りやすい
   - OCPとFactory Methodの連動を体験で理解できる

4. **既存シリーズとの連続性**
   - Mooの基本を活かしつつ新しい題材で差別化できる

### 代替案の選択指針

- **案Bが適しているケース**: 脱出ゲーム系のストーリー性を重視したい場合
- **案Cが適しているケース**: 生活に近い題材で敷居を下げたい場合
