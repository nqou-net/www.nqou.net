# 第3回：通知を受け取るクラスを作ろう

## 概要
ログと実績をそれぞれ独立したクラスに分離。通知を受け取るupdateメソッドを持たせ、責務分離の基礎を学びます。

## 検証結果

### ✅ 構文チェック
```
game.pl syntax OK
```

### ✅ 実行結果
```
=== ダンジョン探索 ===

[LOG] スライムを倒した！
[ACHIEVEMENT] 実績解除: はじめての勝利
[LOG] 薬草を手に入れた！
[LOG] レベルが5になった！
[ACHIEVEMENT] 実績解除: 成長

=== 解除した実績 ===
- はじめての勝利
- 成長
```

## 改善点
第2回の問題を解決するために、以下のアプローチを採用：

1. **クラスの分離**: 
   - `LogObserver`: ログ出力専用
   - `AchievementObserver`: 実績管理専用

2. **共通インターフェース**:
   - 両クラスとも`update`メソッドを持つ
   - イベントを受け取って処理を行う

## 使用されているクラス
- `GameEvent`: イベントを表現（`data`属性を追加）
- `LogObserver`: ログ出力専用クラス
- `AchievementObserver`: 実績管理専用クラス

## 検証者のコメント
責務の分離が実現されました！各Observerが独立しており、テストや保守が容易になっています。まだ明示的なインターフェース定義（Moo::Role）は使っていませんが、`update`メソッドという暗黙的な約束が存在します。次回でこの約束を正式なものにします。
