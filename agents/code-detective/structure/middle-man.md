## 5幕プロット

### 第I幕: 再訪（導入）
**見出し**: 「一週間ぶりの再来」

森川が昼間にLCIを訪問。手には缶コーヒー2本（前回の礼）。前回の「列車事故」を解体した後、1週間かけて全クラスに `handles` を適用したリファクタリングの成果を報告しに来た。

ロックは壁にまた何か貼っている（前回のクラス図ではなく、今度はメソッドの呼び出しグラフ）が、森川を見て「おや、列車事故の——」と思い出す。

森川は自信を持ってリファクタリング結果を見せる。「先週教わった通り、全部 handles で委譲しました。メソッドチェーンは一箇所も残っていません」

ロックが画面を覗き込み、表情が変わる。「……ワトソン君。これは別の問題だ」

### 第II幕: 現場検証（容疑者の浮上）
**見出し**: 「空っぽの部屋」

ロックが問題の `OrderFacade` を指摘する。`customer_name`, `customer_email`, `shipping_zone`, `item_name`, `quantity`, `unit_price`, `total_price` ——すべてのメソッドが `handles` で `order` に丸投げされている。OrderFacade 自身には独自のロジックが一行もない。

森川は反論する。「でも、先週ロックさんが言ったじゃないですか。直接の友人とだけ話せ、見知らぬ者と話すなと。だから全部を1段の委譲にしたんです」

ロックが返す。「友人と話すのは正しい。だが友人に、自分の口で言えることまで代弁させているのが問題だ」

### 第III幕: 推理披露
**見出し**: 「仲介人の審問」

ロックが Middle Man の判定基準を示す。

**Step 1: メソッドを3種類に分類する**
- A. 独自ロジックを持つメソッド（委譲 + 加工・判断・変換があるもの）
- B. 純粋な委譲（ただ渡すだけ）
- C. そのクラスに属すべきロジック

「B が過半数を超えたら、そのクラスは Middle Man だ」

**Step 2: Remove Middle Man — 直接アクセスの許容**
ShippingCalculator が OrderFacade 経由で shipping_zone を取得していたのを、ShippingCalculator が直接 Order を受け取るように変更。OrderFacade の handles を整理し、独自ロジックを持つメソッド（calculate_total 等）だけを残す。

**Step 3: 振り子のバランス**
「LoD 違反（Train Wreck）と Middle Man は振り子の両端だ。完璧な位置は存在しない。プロジェクトの変更頻度と依存の深さで判断する」

### 第IV幕: 解決（テスト）
**見出し**: 「仲介人のいない世界」

テストを実行。OrderFacade から不要な handles を除去し、ShippingCalculator が Order を直接受け取る構造に変更。テスト全パス。

森川が気づく：委譲を減らしたことで、各クラスの責務が読みやすくなった。

### 第V幕: 報告書

探偵の調査報告書フォーマット。

## Before コード概要

```perl
# OrderFacade — handles の過剰使用
# 全メソッドが order 属性への委譲のみ。独自ロジックゼロ。
package OrderFacade;
use v5.36;
use Moo;

has order => (
    is       => 'ro',
    required => 1,
    handles  => [qw(
        customer_name customer_email shipping_zone
        item_name quantity unit_price total_price
    )],
);
```

## After コード概要

```perl
# OrderFacade → 独自ロジック（calculate_total）だけ残す
# ShippingCalculator は Order を直接受け取る
package Order;
use v5.36;
use Moo;

has customer   => (is => 'ro', required => 1, handles => [qw(shipping_zone)]);
has item_name  => (is => 'ro', required => 1);
has quantity   => (is => 'ro', required => 1);
has unit_price => (is => 'ro', required => 1);

sub total_price ($self) {
    return $self->quantity * $self->unit_price;
}

# ShippingCalculator — Order を直接受け取る（OrderFacade を経由しない）
package ShippingCalculator;
use v5.36;
use Moo;

has order => (is => 'ro', required => 1, handles => [qw(shipping_zone)]);

sub calculate ($self) {
    my %rate = (kanto => 500, kansai => 700, other => 1000);
    return $rate{$self->shipping_zone};
}
```
