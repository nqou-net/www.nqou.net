# Plot Outline: Interpreter Pattern

> **舞台**: 来院（コード診療所）
> **患者**: 桐山 蓮（29歳、EC部門テックリード、一人称: 俺）
> **病名**: 急性eval中毒（Acute eval Intoxication）

---

## Scene I: 来院 — 「evalのどこがダメなんすか」

### ト書き
雑居ビルの2階。桐山は知人から渡されたメモを片手に、磨かれたリノリウムの廊下を進む。重厚な鉄扉に「コード診療所」とだけ書かれたプレートを見つけ、半信半疑でドアを引く。

中に入った瞬間、O'Reilly本とマザーボードの山が視界を埋め尽くす。奥のカウンターの向こう——トリプルディスプレイの前に黒いシャツの男が座っている。微動だにしない。

カウンターの横からナナコが現れ、穏やかに迎える。桐山はノートPCを取り出し、管理画面を見せながら「ちゃんと動いてるシステム」を強気で説明し始める。

### 展開ポイント
- **意外性**: 桐山は反省して来たわけではない。「監査のやつが的外れなのを証明してもらう」くらいの気持ちで来ている。evalを自信満々にデモする
- 桐山がノートPCで管理画面を開き、ルール入力欄に `$cart_total >= 5000 && $member_rank eq "gold"` と打ち込んで「ほら、リアルタイムで反映されるんすよ」と得意げに見せる
- ドクターはデモには目もくれず、ソースコードのほうを黙って指差す。eval が書かれた行を
- 桐山は「あ、そこ見ます？ まあ eval っすけど、柔軟にやるにはこれが一番で——」
- ドクターが桐山の言葉を遮るように、片手を上げる（「止まれ」の意）。そして一言

### 主要セリフ
- 桐山:（ドアを引いて入り、O'Reilly本の山を見て）「……病院っていうか、倉庫？」
- ナナコ: 「大丈夫ですよ、ここはコード診療所です。どうぞお座りください」
- 桐山: 「あ、どうも。えっと、監査で引っかかったんすけど、正直、俺のシステムは問題ないと思ってて——ほら、見てもらえますか」（管理画面をデモ）
- 桐山: 「ルール文字列を入力すると、リアルタイムで評価されるんすよ。コード変更なしで。最強じゃないすか？」
- ドクター:（ソースコードの eval 行を指差して）「……これは？」
- 桐山: 「あ、eval っす。条件式を動的に——」
- ドクター:（片手を上げて遮り）「…… **劇薬** だ」
- ナナコ: 「eval は確かに効き目がありますけど、副作用も大きいお薬なんです。用法・用量を間違えると……」

---

## Scene II: 触診 — 「中毒」の自覚

### ト書き
ドクターが桐山のノートPCを手元に引き寄せ、ソースコードをスクロールし始める。eval を含む関数。正規表現による前処理。$@ を握りつぶすエラーハンドリング。ドクターの表情に、かすかな険しさが浮かぶ——ように桐山には見えた。

### 展開ポイント
- ドクターが3つの問題を順に指差す: (1) eval本体、(2) 正規表現による雑な前処理、(3) `$@` の握りつぶし
- 桐山は最初「管理画面だから外部入力じゃないっしょ」と反論するが、ナナコが「管理画面にログインできる人全員がPerl コードを実行できる状態、ということですよ」と返すことで顔色が変わる
- ドクターが端末を操作し、実際にルール入力欄から `system("echo pwned")` のような文字列を入力してみせる（実行はしない。コード上でevalに到達する経路を可視化するだけ）
- 桐山の「最強のシステム」が「最弱のセキュリティ」だったという逆転
- さらに、ドクターは `$@` の握りつぶしを指さして「もう一つ」と言う。エラーが闇に葬られていたことで、不正なルールが「条件不一致」として静かに無視されていた事実

### 主要セリフ
- ドクター:（evalの行を指で叩き）「注射……直接血管に。フィルタなし」
- ナナコ: 「外部から入力された文字列をそのまま実行しているんです。血管に直接何でも注入できてしまう状態、と先生はおっしゃっています」
- 桐山: 「いや、でも管理画面からしか——」
- ナナコ: 「管理画面にログインできる方全員が、Perl のコードを自由に実行できる、ということです。悪意がなくてもタイプミスで事故が起きますよ」
- 桐山:（顔色が変わる）「……あ」
- ドクター:（$@ の握りつぶしを指して）「もう一つ。 **痛覚が死んでいる**」
- ナナコ: 「エラーを握りつぶしているので、異変に気づけないんです。痛みを感じない体は、怪我に気づけませんよね」
- 桐山: 「あれは……ルールがマッチしなかったってことだと思って……」
- ドクター: 「急性eval中毒。 **重症** だ」

## Scene III: 外科手術 — Interpreterパターンの処方

### ト書き
ドクターがキーボードに手を伸ばす。桐山のノートPCではなく、自分のトリプルディスプレイに向かって。新しいファイルを開き、黙々とコードを書き始める。ナナコが桐山の横に立ち、解説を始める。

### 展開ポイント（手術の3ステップ）

#### Step 1: 文法の定義（Expression基底クラス）
- ドクターが `Expression` 基底ロール（Moo::Role）を作成。`interpret($context)` メソッドのインターフェースを定義
- 次に `ComparisonExpression`（比較）、`LiteralExpression`（リテラル）、`VariableExpression`（変数参照）の終端式を実装
- ナナコの通訳: 「ルールの最小単位を『単語カード』にしていくイメージです。『カート合計 >= 5000』なら、『カート合計』カード、『>=』カード、『5000』カードですね」
- 桐山: 「……文字列じゃなくて、オブジェクトにするってことっすか」

#### Step 2: 構文木の組み立て（非終端式 + パーサー）
- ドクターが `AndExpression`、`OrExpression`、`NotExpression` の非終端式を実装。これらは左辺・右辺に他の Expression を持つ
- さらに簡易パーサーを実装し、ルール文字列をトークン化 → Expression オブジェクトのツリーに変換
- ナナコの通訳: 「文字列を分解して、単語カードを組み合わせたツリーを作るんです。『A AND B OR C』なら、ORの下にANDとCがぶら下がって、ANDの下にAとBがぶら下がる——お医者さんで言えば、CTスキャンの画像みたいに構造が見えるようになるんですよ」
- 桐山が eval との違いを直感的に理解する瞬間:「あ、これならevalを通さなくても……各オブジェクトの interpret() を呼ぶだけで結果が出る」

#### Step 3: eval の除去と検証
- ドクターが既存の eval を使ったルール評価関数を、新しい Interpreter ベースの評価に置き換える
- **eval が消える瞬間**。桐山が仰け反る: 「消えた……俺の最強の武器が……」
- ドクターがテストを実行し、全ルールが同じ結果を返すことを確認
- さらにナナコが「試しに不正な式を入れてみましょう」と提案。パーサーが構文エラーを検出して安全に弾く
- 桐山: 「前は eval が黙って食べてたのに、今はちゃんとエラーになる……」

### 主要セリフ
- ドクター:（Expression ロールを書きながら）「……文法」
- ナナコ: 「ルールの文法を、クラス構造で表現するんです。文字列ではなくオブジェクトとして」
- 桐山: 「オブジェクト……。文字列をそのまま eval に渡すんじゃなくて？」
- ナナコ: 「はい。単語カードを作って組み合わせていくイメージです。1枚1枚のカードは安全で、組み合わせ方も安全。注射器（eval）を使わなくても治療できるんですよ」
- ドクター:（And/Or を追加しながら）「……構文木」
- ナナコ: 「カードを木のように組み合わせるんです。CTスキャンみたいに、ルールの構造が見えるようになりますよ」
- ドクター:（evalを削除して）「……排毒完了」
- 桐山: 「あっ——消えた。俺の最強の武器が……」
- ナナコ: 「大丈夫です。もっと安全で、構造が分かる治療法に変わったんですよ。試しに不正な式を入れてみましょうか」
-（不正入力 → パーサーがエラーを返す）
- 桐山: 「前は黙って飲み込んでたのに……ちゃんとエラーになる」
- ドクター:（小さく頷く）

---

## 勘違いコメディシーン（Scene III 内に配置）

### タイミング: Step 2 と Step 3 の間の小休止

ドクターがコーディングの手を止め、立ち上がる。桐山の方に近づき——桐山のノートPCにそっと手を伸ばす。桐山は「俺のPC見てくれるのか……評価してくれるってことか？」と身構えるが、ドクターはノートPCの脇に置いてあった**桐山の缶コーヒー**を無造作に取り上げ、一口飲んで自分の席に戻る。

- 桐山:（やや期待して）「あ、俺のコード見てくれ——」
- ドクター:（缶コーヒーを取って席に戻る）
- 桐山: 「……え、俺のコーヒー？」
- ナナコ:（申し訳なさそうに）「すみません、先生は手の届くところにあるものは全部自分のものだと思うフシがありまして……。新しいのお持ちしますね」
- ドクター:（満足げに缶コーヒーを啜りながらコーディング再開）

---

## Scene IV: 術後経過 — 「46ルール、全部通る」

### ト書き
リファクタリング完了。ドクターがテストを走らせ、46件のルールすべてが正常に評価されることを確認。桐山はモニタに映るテスト結果を見つめている。

### 展開ポイント
- 桐山が「46ルール全部通ってる……」と呟く。eval 時代と同等の結果が、eval なしで実現されている
- ナナコが「新しいルールを追加する場合も見てみましょう」と促す。ドクターが新しい演算子（例: `CONTAINS`）を扱う `ContainsExpression` クラスを数行で追加してみせる。既存コードは一切変更なし
- 桐山が Interpreter パターンの拡張性を体感する:「新しい演算子って、クラス足すだけでいいのか……eval なら正規表現に1行追加で済んだけど、こっちのほうがどこに何があるか分かる」
- ドクターがノートPCをそっと桐山に返す。桐山がそこに eval のなくなったコードを見て、少し寂しそうな、でも晴れやかな顔をする
- ドクターは帰り支度を始める。鞄を持ち上げ、入口のドアに向かう
- 桐山が「ありがとうございました」と深々と頭を下げる
- ドクターは振り返らず、ドアを押して廊下に出ながら一言
- ナナコが最後に「テストもお忘れなく」と微笑み、続いてドアを出る
- 桐山が一人残された診療所の待合スペースで、ノートPCを閉じて立ち上がる。鉄の扉を押して廊下に出る

### 主要セリフ
- 桐山: 「46ルール……全部通ってる」
- ナナコ: 「新しい演算子を追加してみましょうか。先生、`CONTAINS` はどうですか？」
- ドクター:（数行のクラスを書き加え）「……追加完了」
- 桐山: 「え、これだけ？ 既存のコード触ってないっすよね」
- ナナコ: 「新しいお薬（演算子）を棚に追加しただけです。他のお薬には影響しません」
- 桐山: 「……evalのほうが短かったっすけど、こっちのほうが何やってるか分かる」
- ドクター:（鞄を持ち上げながら）「コミットメント」
- ナナコ: 「先生の治療費はテストコードです。新しいルールを追加するたびにテストを書いてくださいね」
- 桐山: 「テスト……ああ、eval時代はテスト書けなかったっすもんね。文字列だから」
- ドクター:（ドアを押して廊下に出ながら、振り返らず）「感謝は、このコードに」
- ナナコ:（微笑んで）「おだいじに。監査の再試験、うまくいきますように」
- 桐山:（一人になった診療所で）「……悔しいけど、最強の武器は eval じゃなかった」

## 舞台設定の整合性チェック

- [x] 来院設定: 桐山が「来て」「帰る」側。ドクターは診療所に居る側
- [x] 桐山の所有物: ノートPC持参。自席・自デスクなし（診療所にはない）
- [x] クリニック固有の小物: O'Reilly本の壁、トリプルディスプレイ、缶コーヒー（待合の自販機または桐山の持参）
- [x] 帰り支度の主語: ドクターが鞄を持つ（往診ではないので帰り支度シーンは不自然 → ドクターが奥の部屋に戻る形式に変更可能だが、ドクター退場→桐山が帰るの順で処理）
- [x] ドアの開閉: 鉄の扉は **外開き** — 入室時「引く」(pull)、退室時「押す」(push)。Scene I で桐山が「引く」、Scene IV で桐山が「押す」で正しい
- [x] 退場シーン: ドクターが先にドアを押して退場→ナナコが続く→桐山が最後にドアを押して廊下に出る

> **Note**: 来院設定だが、ドクターの退場はやや変則的（通常は患者が帰る）。ドクターが「奥の診察室に戻る」のが自然だが、ここでは廊下に出るのではなく「奥の部屋に消える」→桐山がドアを押して退出、という流れに本文執筆時に調整する。
