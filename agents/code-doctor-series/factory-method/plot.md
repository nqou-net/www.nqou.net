# Plot Outline

## Scene 1: The Emergency (Admission)
- **Situation**: 深夜26時、ソーシャルゲーム開発会社のオフィス。ヤマダは明日のイベント開始に向けた「レアモンスター実装」の修正作業中。
- **Incident**: `Game.pm` に `elsif ($type eq 'rare_slime')` を追加した瞬間、なぜか関係ない `Boss` クラスのテストが落ち始める。「なんで！？ 触ってないのに！」と叫ぶヤマダ。
- **Arrival**: 無音でドアが開き、白衣のドクターとナース服のアシスタントが入ってくる。
- **Dialogue**:
  - ドクター: 「……悲鳴。」
  - ヤマダ: 「え？ 僕、叫んでました？」
  - アシスタント: （微笑んで）「いいえ、あなたのGitリポジトリからです。commit logが『助けて』と叫んでいます。」

## Scene 2: The Diagnosis (Examination)
- **Action**: ドクターがヤマダの席に座り、`Game.pm` の `spawn_enemy` サブルーチン（1500行）をスクロールする。
- **Reaction**: ドクターが眉間に深い皺を寄せ、人差し指でデスクを叩く。カツ、カツ、カツ。
- **Dialogue**:
  - ドクター: 「……癒着。壊死寸前。」
  - アシスタント: 「『Gameクラスが全てのモンスタークラスと癒着している。このままでは、新しいモンスターを追加するたびに、母体であるGameクラスが拒絶反応を起こしてショック死する』とのことです。」
  - ヤマダ: 「でも、if文で分岐しないと、何を作るか決められないじゃないですか……」

## Scene 3: The Surgery (Treatment)
- **Plan**: Factory Methodパターンの適用。モンスター生成の責務を `EnemyFactory` に委譲し、具体クラスへの依存を断ち切る。
- **Metaphor**: 「人工子宮（Factory）の設置」。
- **Highlight**:
  - **Before**: 延々と続く `if/elsif` 地獄。
  - **Process**: ドクターがキーボードを叩く速度が上がり、`Game.pm` から肉（コード）が削ぎ落とされていく。
  - **After**: `my $enemy = EnemyFactory->create($type);` の1行だけ残る。
  - **Reaction**: ヤマダ「えっ、これだけ？ これで動くんですか？」

## Scene 4: The Prognosis (Recovery)
- **Confirmation**: 新しいモンスター「メタルスライム」を追加設定だけで実装してみる。`Game.pm` を一切触らずに動作することを確認。
- **Comedy (The Misunderstanding)**:
  - ドクターがヤマダの顔をじっと見つめ、手を伸ばす。
  - ヤマダ: （心の声：『よくやった』と握手を求めているのか！？ 感動だ！）
  - ヤマダが手を差し出す。
  - ドクターはヤマダの手を無視し、ヤマダの眼鏡のズレを直す。
  - ドクター: 「軸が、ブレている。」
  - アシスタント: 「視点が定まっていないと、コードの設計もブレるそうです。」
  - ヤマダ: （空を切った手をゆっくり下ろす）「……はい。」
