# Patient Profile: Bridge Pattern

## Patient Profile

- **Name**: 帳簿の魔術師・ミサキ (38)
- **Role**: 元経理部エクセル職人 → 社内SE（異動1年目）。社内レポート出力システムの開発担当。
- **Personality**: 几帳面で表が大好き。Excel時代に培った「動けば正義」の精神で、力技の条件分岐を量産する。数字には強いがオブジェクト指向は「何語？」状態。「ちゃんと動いてるんだから大丈夫でしょ」が口癖。
- **Background**: 経理部で10年、Excelマクロだけで部門の業務を回してきた伝説の職人。「プログラミングできるならSEもできるでしょ」と異動を命じられ、見よう見まねでPerlを書き始めた。ExcelのIF関数ネストの感覚がそのままコードに反映されている。
- **First Person**: 「私」
- **Chief Complaint（主訴）**: 「ちゃんと動いてるんですが、部長が『新しい形式追加して』って言うたびに胃が痛くなるんです」

## Medical Chart

- **Symptom**: 部門（営業・経理・人事）×出力形式（PDF・CSV・HTML）の全9パターンを1つの巨大関数内のif-elsif分岐で実現。200行超のネスト。新部門・新形式の追加で分岐が指数的に増殖し、修正のたびにデグレが発生。
- **Metaphor**: 「慢性条件分岐瘴毒——ロジックの空気が分岐の瘴気に汚染され、どこにメスを入れても毒素（バグ）が噴き出す。しかも臓器（部門ロジック）と血管（出力形式）が完全に癒着しており、分離なしには手が出せない」
- **Diagnosis**: 二軸癒着症候群（Biaxial Adhesion Syndrome）——抽象と実装の未分離による慢性的構造障害
- **Cure**: Bridgeパターンによる癒着分離手術。部門ロジック（Abstraction）と出力形式（Implementation）を独立したクラス階層に分離し、has-a関係で再接合する。
