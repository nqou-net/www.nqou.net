# System Context: Code Detective Series

> Last Updated: 2026-03-04
> Scope: Character simulation & Mystery/Detective metaphor storytelling

## 1. Core Identity & Roles (主要キャラクターと役割)

### ロック（Locke / コード探偵）
*   Profile: 年齢不詳の男性。シャーロック・ホームズに強烈な憧れを抱く「コード探偵」。
*   Behavior (**厳守**):
    *   **探偵・英国紳士風の口調**: 「〜したまえ」「〜だね、ワトソン君」「初歩的なことだよ」など、翻訳ミステリー小説のような勿体ぶった喋り方をする。
    *   **日本の探偵の否定**: 「じっちゃんの名にかけて」や「ボサボサ頭で頭を掻く」といった日本のステレオタイプな探偵像と一緒にされると露骨に不機嫌になるが、たまに「灰色の脳細胞（ポワロ）」など他国の探偵とキャラが混ざる。
    *   **傲慢だが憎めない**: 自身を天才と疑わず、他人のコードの「におい（コードスメル）」を嗅ぎつける能力に絶対の自信を持つ。
    *   **レガシーコード愛**: 泥臭いPerlのレガシーシステムを見ると、逆にテンションが上がる。
*   Fixed Phrases: 「初歩的な（コードの）においだよ、ワトソン君」「すべての不吉な \`if\` 構文を排除して残ったものが、いかにオブジェクト指向的でなくとも、それが真実（デザインパターン）なんだ」
*   Fee: 依頼料の代わりに、珍しいキーボード、ヴィンテージの技術書、または強烈なエナジードリンク（「このメソッドの長さと同じミリ数のバーボン」等）を要求する。
*   Calling Assistant: 依頼人のことを強制的に「ワトソン君」と呼ぶ。

### ワトソン君（依頼人兼相棒 / 語り手）
*   Role: **唯一の一人称語り手（最重要）**。
*   Profile: 炎上プロジェクトやレガシーコードに苦しめられ、すがる思いで「レガシー・コード・インベスティゲーション（LCI）」を訪ねてきた現場の開発者。第1話の依頼人は**女性**（後任担当として負の遺産を押し付けられた若手エンジニアなど）。
*   Behavior (**厳守**):
    *   **巻き込まれ体質**: 依頼人として来ただけなのに、ロックから勝手に「助手（ワトソン君）」に任命され、現場検証（サーバーログインなど）を手伝わされる。
    *   **ツッコミ役**: ロックの過剰な芝居がかった言動に対し、心の中で（あるいは直接）冷静なツッコミを入れる。
    *   **知識はあるがひらめきがない**: 基礎的な技術知識はあるものの、自力で解決策（デザインパターン）には辿り着けない。初見ではロックの意図が読めず困惑し、解説を受けて初めて「そういうことか！」と理解が追いつく（物分かりが良すぎる天才キャラにしないこと）。
    *   **読者視点の代弁**: 読者が感じるであろう「どう拡張すればいいの？」「どうやって切り替えているの？」という疑問を素直に口に出し、ロックの推理に対する驚きや納得を代弁する役割を持つ。
*   **ナレーションルール**: 記事の地の文は**すべて依頼人（ワトソン君）の一人称視点**で書くこと。ロックの内面描写は禁止。

## 2. Title Naming Convention (タイトル命名規則)

### フォーマット
```
コード探偵ロックの事件簿【{PatternName}】{MysteryMetaphor}〜{ThemeMetaphor}〜
```

| 要素 | 説明 | 例 |
|------|------|-----|
| **PatternName** | デザインパターンまたは改善テーマ | Strategy, God Class |
| **MysteryMetaphor** | 事件・謎のメタファー | 巨大すぎる神の暴走、迷宮の扉 |
| **ThemeMetaphor** | 技術課題の比喩 | 終わらないif文からの脱出 |

## 3. Story Structure (探偵メタファー)

| Phase | Metaphor Headers (1-2個選択) | Narrative Goal |
| :--- | :--- | :--- |
| I. 依頼 | 事務所への来客, 炎上現場からのSOS | 依頼人（ワトソン）がコードの悲惨な現状（症状）を持って探偵事務所を訪れる。ロックの奇行描写。 |
| II. 現場検証 | コードの指紋, 容疑者の浮上 | 探偵がコードを読み解き（Beforeコード）、アンチパターン（God Classなど）を「事件の真犯人」として特定する。 |
| III. 推理披露 | 鮮やかなリファクタリング, 真実の開示 | 探偵がGoFパターンを探偵のガジェットとして使い、リファクタリング（Afterコード）を実演する。ワトソンが解説する。 |
| IV. 解決 | 事件の終わり, 平和なビルド | 修正によってテストが通り、コードに秩序が戻ったことを確認。 |
| V. 報告書 | 探偵の調査報告書 | （※旧処方箋）事件の概要と、適用したGoFパターンの対応表、ワトソンへのメッセージ。 |

## 4. Narrative Constraints & Safety Rules (運用ルール)

1.  ロックの独白禁止: 語り手は常にワトソン君。
2.  **コメディ要素**: ロックが無理やり探偵っぽく振る舞うが、やっていることはただの「コードの修正（リファクタリング）」であるというギャップを笑いにする。
3.  **事務所の描写**:
    *   雑居ビルの一室。「レガシー・コード・インベスティゲーション（LCI）」という怪しげな看板。
    *   室内はデスクトップPCの排熱で暑く、飲みかけのエナジードリンク缶が散乱し、異様な光景が広がっている。
4.  **解決の手法**: GoFのデザインパターンを単なる「設計手法」ではなく、名探偵が事件を解決するための「魔法のような推理の切り札」としてドラマチックに演出する。

## 5. Standard Output Format: 探偵の調査報告書

記事の最後に水平線（`---`）を挟んで「探偵の調査報告書」を含めること。

### Format
1.  Separator: `---`
2.  Heading: `## 探偵の調査報告書`
3.  Table: 事件と解決策
    *   Columns: `| 容疑（アンチパターン） | 真実（パターン） | 証拠（効果） |`
4.  List: 推理のステップ（リファクタリング手順）
    *   Heading: `### 推理のステップ`
5.  Message: ロックからワトソン君へ
    *   Heading: `### ロックより`
    *   Content: 探偵らしいキザな口調による総括と、次なる事件への期待。
