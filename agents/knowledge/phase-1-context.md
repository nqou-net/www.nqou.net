## 1. 患者（Patient）の設計ガイド

### 役割
- **唯一の語り手**: 完成記事は全て患者の一人称視点で書かれる
- このフェーズでは**ペルソナと一人称**を決定する

### 一人称の選択肢

| 一人称 | 適するペルソナ |
|--------|---------------|
| 僕 | 真面目な若手、控えめな中堅 |
| 私 | フォーマル、女性、慎重派 |
| 俺 | 自信家、ベテラン、フリーランス |
| 自分 | 体育会系、元SI |

### ペルソナの多様性

**バリエーション例:**
- 焦るフリーランス（締め切りに追われて設計を疎かに）
- 傲慢な古参（「動けばいい」主義）
- 悩める新人（基礎を学ばずコピペ実装）
- 完璧主義の中堅（過剰設計で身動き取れず）

## 2. 医療メタファー変換

### 技術問題 → 医療用語

| 技術的問題 | 医療メタファー |
|-----------|---------------|
| 密結合 | 癒着 |
| 巨大クラス | 肥大化した臓器 |
| 重複コード | 転移 |
| 条件分岐の乱立 | 腫瘍の増殖 |
| 依存関係の複雑化 | 血管の絡まり |

### 治療 → パターン適用

| デザインパターン | 医療的処置 |
|-----------------|-----------|
| Strategy | バイパス手術 |
| Factory Method | 癒着分離手術 |
| Singleton | 人格統合治療 |
| Adapter | 血管変換手術 |
| Decorator | 移植・接木 |

## 3. 出力フォーマット

生成物は `agents/code-doctor-series/<slug>/profile.md` に保存:

# Medical Chart
- Symptom: [技術的症状]
- Metaphor: 「[医療メタファー]」
