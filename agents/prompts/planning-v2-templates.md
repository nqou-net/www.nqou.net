# /planning-v2 プロンプトテンプレート

> ワークフロー: [planning-v2.md](../../.agent/workflows/planning-v2.md)

---

## 基本形（最小）

```
/planning-v2 <テーマ>
```

**例：**
```
/planning-v2 Perlのデザインパターン入門
```

---

## 詳細形（推奨）

```
/planning-v2

## テーマ
<何についての記事か>

## コンテンツタイプ
<シリーズ記事 / 単体記事 / チュートリアル / リファレンス>

## 対象読者
<スキルレベル、前提知識>

## ゴール
<読者が得られる成果>
```

**例：**
```
/planning-v2

## テーマ
Mooを使ったPerlオブジェクト指向プログラミング

## コンテンツタイプ
シリーズ記事

## 対象読者
- Perlの基本文法は理解している
- OOPの概念は知っているが実践経験が少ない

## ゴール
Mooを使って実用的なクラス設計ができるようになる
```

---

## フル指定形（複雑なテーマ向け）

```
/planning-v2

## テーマ
<何についての記事か>

## コンテンツタイプ
<シリーズ記事 / 単体記事 / チュートリアル / リファレンス>

## 対象読者
- <スキルレベル>
- <前提知識>

## ゴール
<読者が得られる成果>

## スコープ
### 含める内容
- <含めたいトピック1>
- <含めたいトピック2>

### 除外する内容
- <除外したいトピック1>
- <除外したいトピック2>

## 既存調査
<あれば content/warehouse/ のファイルパス>

## 参考にしたい記事/パターン
<あれば agents/structure/ のファイルパス>
```

---

## コンテンツタイプ別のヒント

| タイプ | ポイント |
|--------|----------|
| **シリーズ記事** | 1記事1概念を意識、難易度勾配を指定 |
| **単体記事** | スコープを明確に、広げすぎない |
| **チュートリアル** | 成果物を具体的に、所要時間の目安を提示 |
| **リファレンス** | 網羅性 vs 実用性のバランスを指定 |

---

## 追加オプション

必要に応じてプロンプトに追加できる項目：

| 項目 | 用途 |
|------|------|
| `## 差別化ポイント` | 競合記事との違いを強調したい点 |
| `## 制約` | 文字数、回数、技術スタック等の制限 |
| `## 参考URL` | 参考にしたい外部記事のURL |
| `## 避けたいパターン` | 過去の失敗例、やりたくないこと |
