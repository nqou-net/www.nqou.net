new <title></code> と書いてあったのを見て、もしかして、と思った。<br>で、調べてみたところ案の定、って感じ。</p> <h2 id="コマンド体系"><a href="#コマンド体系" class="headerlink" title="コマンド体系"></a>コマンド体系</h2><p>ところで、最近のコマンドはサブコマンド形式が多い。</p> <p>普段お世話になっている <code>brew</code> もそうだし、この <code>hexo</code> もそう。</p> <p>また、サブコマンドを入力しなかった時に使い方がでる、というスタイルや、 <code>コマンド help サブコマンド</code> のようにしてサブコマンドのヘルプを呼び出す、というのもだいたい共通している。</p> <p>また、上の方では <code>hexo help new</code> として結果を書いているが、 <code>hexo new</code> でも同じ内容のヘルプが出る。</p> <p>ターミナル上で操作をしていると、新しいコマンドもとりあえず同じように使ってみる、ということがあると思う。</p> <p>取扱説明書を熟読してから家電を使う人はいない。</p> <p>コマンドもそうだと思う。</p> <p>ヘルプを見なくても使えるコマンドは素晴らしいコマンドだ。</p> <p><code>plenv</code> のような、 <code>**env</code> は沢山ある。</p> <p><code>anyenv</code> を使うと、そのような <code>**env</code> が気軽に使えるようになる。</p> <p>しかし、時々コマンド体系が違うものがあって嫌な感じになる。</p> <p>例えば <code>goenv</code> に関しては、インストール可能なバージョンを確認しようとして <code>goenv install -l</code> とするとエラーになる。</p> <p>で、 <code>goenv install</code> ではどうかというと、以下のようになる。</p>
