# Code Doctor Narrative Skill

このスキルは、「コードドクター」シリーズにおいて、読者を惹きつける「素晴らしい物語」を生成するための技術ガイドラインです。

## 1. 医療メタファーの等価変換技術

読者が「なるほど」と膝を打つメタファーを作成するための思考プロセスです。

1. **技術的特徴の抽出**:
   - 例: Singletonパターン → 「世界に一つしか存在してはいけない」
   - 例: Observerパターン → 「変化があったら即座に通知する」

2. **医療/病院コンテキストへのマッピング**:
   - Singleton → 「心臓（一つしかない）」あるいは「院長（一人しかいない）」
   - Observer → 「ナースコールシステム」あるいは「バイタルモニタリング」

3. **ズレの調整**:
   - メタファーが深刻すぎないか？（コードの修正は「死」ではない）
   - コミカルな要素を含める余地があるか？

## 2. 「勘違いコメディ」の構築レシピ

ドクターと患者の間で発生する「すれ違い」を意図的に設計します。

**基本構造**:
1. **Trigger (行動)**: ドクターが純粋に技術的な理由、または無意識の癖で何かをする。
   - 例: コーヒーを差し出す（単に自分が飲みたかったがついでに入れた）。
2. **Misinterpretation (誤解)**: 患者（語り手）がそれを情緒的に解釈する。
   - 例: 「なんて優しい先生なんだ…僕の緊張をほぐそうとしてくれている」
3. **Reaction (反応)**: 助手がそれを察知し、あえて訂正せずに流す、あるいは微笑む。
4. **Resolution (オチ/未解決)**: ドクターは自分が感謝された理由がわからず、不思議そうな顔をする。

**注意点**:
- ドクターを「バカ」に見せないこと。彼はあくまで「技術の天才」であり、「情緒に疎い」だけである。
- 患者を「愚か」に見せないこと。誰もが陥る心理的なバイアスとして描く。

## 3. シーン構成のペース配分 (Pacing)

記事全体を1つのドラマとして構成します。

- **I. 導入 (15%)**: 
  - **Tempo**: 速い。緊迫感。
  - **Key**: 患者の「困り果てた様子」と、診療所の「非日常的な空気感」の対比。
- **II. 検査 (25%)**: 
  - **Tempo**: 遅い。静寂。
  - **Key**: ドクターの鋭い視線（コードレビュー）。沈黙の緊張感。
- **III. 処置 (40%)**: 
  - **Tempo**: リズミカル。知的興奮。
  - **Key**: コードが鮮やかに変わっていく快感（カタルシス）。ドクターの短い指示と助手の流れるような解説。
- **IV. 予後 (20%)**: 
  - **Tempo**: 穏やか。温かみ。
  - **Key**: 安堵感。勘違いコメディの回収（または放置）。

## 4. Sub-Agent Roles

このスキルを使用する際、以下のような「人格」を切り替えて執筆すると効果的です。

- **The Screenwriter (脚本家)**: プロットの骨組み、伏線、オチを担当。
- **The Translator (翻訳家)**: 技術用語をセリフ（医療用語）に変換する際、自然な日本語を選択。
- **The Director (演出家)**: 「ここでドクターに溜息をつかせる」「ここで助手に目配せさせる」といったト書き（描写）を追加。
