# Series Author (汎用統合執筆スキル)

## 概要
「シリーズマニフェスト（設定ファイル）」を読み込むことで、指定された世界観（メタファー）、キャラクター（ペルソナ）、プロット構成に従って技術解説記事を生成します。
このスキル一つで「コード探偵」や「コードドクター」を含む、あらゆるシリーズに対応可能です。

## 使用方法（事前確認）

ユーザーから以下の情報を受け取ってください。
1. **シリーズ名（マニフェスト名）**: 必須（例: `code-detective`, `code-doctor`）
2. **テーマ / パターン名**: 必須（例: `Strategy`, `God Class`）
3. **slug**: テーマ名のケバブケース（例: `strategy-pattern`）

## 実行チェイン（汎用ワークフローの実行）

以下のPhase 0〜4のワークフローを順番に実行します。
**各Phaseの終了時に、必ず次のPhaseへ進む前にユーザーに確認または報告を行ってください。**

### Phase 0: 企画・設計
- 実行: `.agent/workflows/phase-0-planning.md`
- 目的: シリーズマニフェストの読み込み、テーマ設定、語り部プロファイルの決定、タイトル案の作成。
- **停止条件**: ユーザーに企画案（タイトル、フック、語り部設定）を提示し、承認を得るまで次に進まない。
- **選択肢の提示**: タイトル案など選択肢が決まっている場合は、テキストで羅列せず `AskUserQuestion` ツールを使って選択式で提示すること。

### Phase 1: プロファイル・プロット作成
- 実行: `.agent/workflows/phase-1-profiling.md`
- 目的: 承認された企画に基づき、詳細なプロファイルとプロット（構造案）をファイルとして生成。
- 出力: `agents/structure/<シリーズ名>-<slug>.md`

### Phase 2: コード実装・検証
- 実行: `.agent/workflows/phase-2-coding.md`
- 目的: Before/Afterのコードを実装し、テストを書いて動作検証を行う。
- 出力: `agents/tests/<シリーズ名>-<slug>/` 配下のコードとテスト。
- **停止条件**: テストが完全にパスするまで修正を繰り返す。警告も残さない。

### Phase 3: 記事本文執筆
- 実行: `.agent/workflows/phase-3-writing.md`
- 目的: マニフェスト（ペルソナ・メタファー・フォーマット）の制約に従い、検証済みコードを用いて記事本文を執筆する。
- 出力: `content/post/YYYY/MM/DD/<slug>.md`

### Phase 4: 記事レビュー・修正
- 実行: `.agent/workflows/phase-4-review.md`
- 目的: 執筆した記事がマニフェストのルール（一人称の統一、キャラクターの言動、メタファーの一貫性、フォーマット）を満たしているか厳密にチェックし、問題があれば修正する。
- **停止条件**: すべてのチェック項目をクリアするまで修正を繰り返す。

## 品質ゲート（絶対ルール）
1. Phase 0 でユーザーの承認を得ずに Phase 1 へ進まない。
2. Phase 2 でテストが通っていないコードを記事に掲載しない。
3. 記事の地の文は、必ずマニフェストで指定された「語り部」の一人称で書く。神の視点や主人公の内面描写は絶対に行わない。
