# プロット構成: ミステリー（5幕構成）

## 概要
探偵シリーズにおける記事全体の流れ（プロット構造）を定義します。
技術的な解説を、探偵が事件を解決する5つのフェーズに沿って展開します。

## 構成案

| フェーズ | 見出しのメタファー例（1〜2個選択） | ナラティブの目標（何を描くか） |
| :--- | :--- | :--- |
| **I. 導入（依頼）** | 事務所への来客, 炎上現場からのSOS | 依頼人（ワトソン）がコードの悲惨な現状（症状）を持って探偵事務所を訪れる。または探偵が現場に赴く。探偵の奇行描写を含める。 |
| **II. 現場検証** | コードの指紋, 容疑者の浮上 | 探偵が既存のコード（Beforeコード）を読み解き、アンチパターン（God Classなど）を「事件の真犯人」として特定する。 |
| **III. 推理披露** | 鮮やかなリファクタリング, 真実の開示 | 探偵がGoFパターンを探偵のガジェットとして使い、リファクタリング（Afterコード）を実演する。ワトソンが読者目線で解説・質問する。 |
| **IV. 解決** | 事件の終わり, 平和なビルド | 修正によってテストが通り、コードに秩序が戻ったことを確認する。 |
| **V. 報告書** | 探偵の調査報告書 | 事件の概要と、適用したGoFパターンの対応表、ワトソンへのメッセージをまとめる。 |
