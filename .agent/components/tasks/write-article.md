# タスク: 統合記事の執筆 (Write Article)

与えられた構造案、コード例、および各種設定（Role, Persona, Format）に基づき、1つの統合された技術記事を執筆してください。

## 実行ステップ

### Step 1: 導入部の作成
記事冒頭（Frontmatterの直後）に以下を含めてください。
1. **フック**: 読者の課題・悩みを提示（物語の導入）
2. **解決策の提示**: この記事で何ができるようになるか
3. **習得内容一覧**: パターンや技術の表形式まとめ
4. **対象読者**: 前提知識・推奨スキル
5. **技術スタック**: 使用する言語・ライブラリ

### Step 2: 本文の作成（章構成）
構造案で指定された各章を、物語の進行に合わせて展開してください。
各章は基本的に以下の流れ（またはそれに準ずる論理的展開）を含みます。
- **動くが脆いコード（問題の提示）**
- **破綻・限界の発生（アンチパターン・症状の露呈）**
- **パターンの導入（解決策の提示）**
- **完成したコード（リファクタリング結果）**
- **解説・ポイント（技術的な学び）**

*注意: 展開の方法や章のタイトルは、読み込んでいる Persona のルール（医療メタファー、探偵メタファーなど）に強く依存します。Persona の指示を最優先してください。*

### Step 3: まとめ章の作成
最終章として以下を含めてください。
1. 全体の振り返りと学んだパターンのまとめ
2. パターン間の関係性（Mermaid図などを活用）
3. 次のステップ・発展的な学習への誘導
4. Persona固有の締めくくり（「退院指導」「探偵の調査報告書」など）

### Step 4: 品質チェック
執筆完了前に、自身で以下の点を確認してください。
- Frontmatter のタグに禁止事項が含まれていないか
- 地の文の視点（一人称など）が Persona に合致しているか
- 技術的な解説とコードの整合性が取れているか
