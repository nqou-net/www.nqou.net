# Phase 6: レビュー・最終確認

> 前: `/oop-fp-hybrid-visual`

---

## Step 1: コード検証

### テスト再実行

// turbo
```bash
cd agents/tests/oop-fp-hybrid-design && prove -l t/
```

### 記事内コードとの一致確認

- 記事内のコード例が `agents/tests/oop-fp-hybrid-design/` のコードと一致するか確認
- 差分がある場合は記事を修正

---

## Step 2: 品質レビュー

### 構造確認

- [ ] 連載構造案のストーリーに沿っているか
- [ ] 各回/章が1概念を守っているか
- [ ] ストーリー構成: 動く→破綻→パターン導入→完成

### SOLID原則確認

- [ ] SRP/OCP違反の指摘が具体的か
- [ ] 「なぜダメか」が明確か

### 技術的正確性

- [ ] `use v5.36;` がすべてのコード例にあるか
- [ ] Moo, Types::Standard, List::Util の使用が正しいか
- [ ] Perl v5.36+ の機能活用が適切か

---

## Step 3: 文体チェック

- [ ] 本文: です・ます調
- [ ] 箇条書き: だ・である調、末尾句点なし
- [ ] 強調表現（**太字**）が使用されていないか
- [ ] 遊び心ある表現があるか

---

## Step 4: 誤字脱字チェック

以下のツールで確認:

// turbo
```bash
# textlint がある場合
npx textlint content/post/YYYY/MM/DD/*.md
```

---

## Step 5: Mermaid図・挿絵確認

- [ ] 第2,5,6,7,8回/章に Mermaid 図があるか
- [ ] 第5,8回/章の図は必須（FCISレイヤー図、全体アーキテクチャ図）
- [ ] 各記事に挿絵が1つ以上あるか
- [ ] アイキャッチ画像が frontmatter で設定されているか

---

## Step 6: frontmatter確認

- [ ] `title`: SEO最適化版タイトル
- [ ] `description`: 各回用メタディスクリプション（150-160文字）
- [ ] `tags`: 推奨タグ（`perl` を含む）
- [ ] `image`: アイキャッチ画像パス
- [ ] `draft: true`: 公開前は true

---

## Step 7: 最終確認

### 連載版のみ

- [ ] シリーズ間リンクが正しいか
- [ ] 前回振り返り・次回予告があるか
- [ ] 目次記事が作成されているか
- [ ] `PLANNING_STATUS.md` に追加されているか

### 統合版のみ

- [ ] 章間の接続が滑らかか
- [ ] 目次（記事内）があるか

---

## 完了

すべてのチェックが完了したら:

1. `draft: false` に変更（または PR 提出）
2. ユーザーに完了を通知
