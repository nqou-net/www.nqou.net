# Phase 1: 準備（汎用）

> 前: `/unified-article-from-series` または シリーズ固有ワークフロー | 次: `/series-unified-code`

---

## 前提条件

このワークフローを開始する前に、以下の情報が必要です：

- **構造案ファイルパス**: `agents/structure/<slug>.md`
- **出力形式**: 連載版 / 統合版

---

## Step 0: 案選定（ユーザー選択）

構造案に複数の案がある場合、最初にユーザーに選択を確認します。

### 0.1 比較表の提示

| 観点 | 案A | 案B | 案C |
|------|-----|-----|-----|
| 特徴 | ... | ... | ... |
| メリット | ... | ... | ... |
| デメリット | ... | ... | ... |
| 推薦度 | ★★★ | ★★☆ | ★☆☆ |

### 0.2 推薦案の説明

- 推薦する案とその理由を明確に説明
- ユーザーの状況に合わせたカスタマイズ提案

### 0.3 ユーザー選択の記録

構造案ファイルに選定結果を追記:

```markdown
## 選定結果

- **採用案**: 案X
- **選定日**: YYYY-MM-DD
- **選定理由**: ...
```

### 0.4 ステータス更新

PLANNING_STATUS.md を更新:

1. ステータスを「✏️ 採用済み」に変更
2. 採用案（案A/B/C等）を記録
3. 更新日を本日に更新

> [!NOTE]
> 構造案が1案のみの場合、このステップはスキップ可能です。

---

## Step 1: 構造案の読み込み

構造案ファイルを読み込み、以下を確認します：

1. シリーズ名・タイトル
2. 全体構成（全N回/N章）
3. 技術スタック
4. 対象読者

// turbo
```bash
# 構造案ファイルが存在することを確認
ls -la agents/structure/
```

---

## Step 2: 公開日時の決定

ユーザーに以下を確認：

### 統合版（1記事）の場合

- **公開日時**: 1つの日時を指定

### 連載版（複数記事）の場合

- **連載開始日**: 最初の記事の公開日時
- **公開間隔**: 記事間の間隔（例: 1日おき、2日おき）

// turbo
```bash
# 現在時刻を取得（日時フォーマットの参考）
date +"%Y-%m-%dT%H:%M:%S+09:00"
```

---

## Step 3: 出力ディレクトリの準備

公開日時に基づきディレクトリを作成：

```bash
# 例: 2026-02-01の場合
mkdir -p content/post/2026/02/01
```

---

## Step 4: 設定の記録

以下の情報を記録（task.mdに保存推奨）：

| 項目 | 値 |
|------|-----|
| 構造案ファイル | `agents/structure/<slug>.md` |
| 採用案 | 案A / 案B / 案C など |
| 出力形式 | 統合版 / 連載版 |
| 公開日時 | YYYY-MM-DDTHH:MM:SS+09:00 |
| テストディレクトリ | `agents/tests/<slug>/` |

---

## 完了後

→ `/series-unified-code` へ進む

---

## 注意事項

- 構造案のフォーマットは `agents/structure/` 内の既存ファイルを参考
- 技術スタックの依存関係は事前に確認
- 公開日時は未来日付でもOK（Hugo `-F` フラグで確認可能）
