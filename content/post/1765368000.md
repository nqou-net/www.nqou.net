## コード生成エージェントの基本

### コード生成の3つのモード

コード生成エージェントは、用途に応じて3つのモードで動作します：

**1. 新規作成モード**
```
入力: 要件と仕様
↓
出力: 完全な新規ファイル
```

**2. 拡張モード**
```
入力: 既存ファイル + 追加したい機能
↓
出力: 機能追加されたコード
```

**3. リファクタリングモード**
```
入力: 既存コード + 改善方針
↓
出力: リファクタリング後のコード
```

### 効果的なプロンプトの構成要素

高品質なコードを生成するプロンプトには、以下の要素を含めます：

```
1. 目的: 何を実装するか
2. 要件: 機能要件と非機能要件
3. 制約: コーディング規約、使用ライブラリ
4. コンテキスト: 既存コードとの関係
5. 期待する出力: ファイル構成、命名規則
```

## 実践例1: REST API エンドポイントの生成

### ステップ1: 詳細なプロンプト作成

```
タスク: ユーザー管理 API エンドポイントを実装

要件:
- GET /api/users - ユーザー一覧取得（ページネーション対応）
- GET /api/users/:id - ユーザー詳細取得
- POST /api/users - ユーザー新規作成
- PUT /api/users/:id - ユーザー更新
- DELETE /api/users/:id - ユーザー削除

技術仕様:
- Node.js + Express
- PostgreSQL（pg ライブラリ使用）
- async/await でエラーハンドリング
- Joi でリクエストバリデーション
- Winston でロギング

ファイル構成:
- src/routes/users.js - ルーティング定義
- src/controllers/userController.js - ビジネスロジック
- src/models/User.js - データモデル
- src/validators/userValidator.js - バリデーション

コーディング規約:
- インデント: スペース2つ
- アロー関数を優先
- JSDoc コメント必須
- エラーは適切にログ記録
```

### ステップ2: コード生成エージェント実行

エージェントパネルで実行すると、以下のファイルが生成されます：

**src/routes/users.js**
```javascript
/**
 * ユーザー管理 API ルーティング
 * @module routes/users
 */

const express = require('express');
const router = express.Router();
const userController = require('../controllers/userController');
const { validateUser, validateUserId } = require('../validators/userValidator');
const asyncHandler = require('../middleware/asyncHandler');

/**
 * @route GET /api/users
 * @desc ユーザー一覧を取得
 * @access Public
 */
router.get('/', asyncHandler(userController.getUsers));

/**
 * @route GET /api/users/:id
 * @desc ユーザー詳細を取得
 * @access Public
 */
router.get('/:id', validateUserId, asyncHandler(userController.getUserById));

/**
 * @route POST /api/users
 * @desc 新規ユーザーを作成
 * @access Public
 */
router.post('/', validateUser, asyncHandler(userController.createUser));

/**
 * @route PUT /api/users/:id
 * @desc ユーザー情報を更新
 * @access Public
 */
router.put('/:id', validateUserId, validateUser, asyncHandler(userController.updateUser));

/**
 * @route DELETE /api/users/:id
 * @desc ユーザーを削除
 * @access Public
 */
router.delete('/:id', validateUserId, asyncHandler(userController.deleteUser));

module.exports = router;
```

**src/controllers/userController.js**
```javascript
/**
 * ユーザー管理コントローラー
 * @module controllers/userController
 */

const User = require('../models/User');
const logger = require('../utils/logger');

/**
 * ユーザー一覧を取得（ページネーション対応）
 * @param {Object} req - Express request object
 * @param {Object} res - Express response object
 */
const getUsers = async (req, res) => {
  try {
    const page = parseInt(req.query.page) || 1;
    const limit = parseInt(req.query.limit) || 10;
    const offset = (page - 1) * limit;

    const { users, total } = await User.findAll({ limit, offset });

    logger.info('Users retrieved successfully', {
      page,
      limit,
      total,
      returned: users.length
    });

    res.json({
      data: users,
      pagination: {
        page,
        limit,
        total,
        pages: Math.ceil(total / limit)
      }
    });
  } catch (error) {
    logger.error('Failed to retrieve users', {
      error: error.message,
      stack: error.stack
    });
    throw error;
  }
};

/**
 * ユーザー詳細を取得
 * @param {Object} req - Express request object
 * @param {Object} res - Express response object
 */
const getUserById = async (req, res) => {
  try {
    const { id } = req.params;
    const user = await User.findById(id);

    if (!user) {
      return res.status(404).json({
        error: {
          message: 'User not found',
          code: 'USER_NOT_FOUND'
        }
      });
    }

    logger.info('User retrieved successfully', { userId: id });

    res.json({ data: user });
  } catch (error) {
    logger.error('Failed to retrieve user', {
      userId: req.params.id,
      error: error.message,
      stack: error.stack
    });
    throw error;
  }
};

/**
 * 新規ユーザーを作成
 * @param {Object} req - Express request object
 * @param {Object} res - Express response object
 */
const createUser = async (req, res) => {
  try {
    const userData = req.body;
    const user = await User.create(userData);

    logger.info('User created successfully', {
      userId: user.id,
      email: user.email
    });

    res.status(201).json({ data: user });
  } catch (error) {
    logger.error('Failed to create user', {
      error: error.message,
      stack: error.stack
    });
    throw error;
  }
};

/**
 * ユーザー情報を更新
 * @param {Object} req - Express request object
 * @param {Object} res - Express response object
 */
const updateUser = async (req, res) => {
  try {
    const { id } = req.params;
    const userData = req.body;

    const user = await User.update(id, userData);

    if (!user) {
      return res.status(404).json({
        error: {
          message: 'User not found',
          code: 'USER_NOT_FOUND'
        }
      });
    }

    logger.info('User updated successfully', { userId: id });

    res.json({ data: user });
  } catch (error) {
    logger.error('Failed to update user', {
      userId: req.params.id,
      error: error.message,
      stack: error.stack
    });
    throw error;
  }
};

/**
 * ユーザーを削除
 * @param {Object} req - Express request object
 * @param {Object} res - Express response object
 */
const deleteUser = async (req, res) => {
  try {
    const { id } = req.params;
    const deleted = await User.delete(id);

    if (!deleted) {
      return res.status(404).json({
        error: {
          message: 'User not found',
          code: 'USER_NOT_FOUND'
        }
      });
    }

    logger.info('User deleted successfully', { userId: id });

    res.status(204).send();
  } catch (error) {
    logger.error('Failed to delete user', {
      userId: req.params.id,
      error: error.message,
      stack: error.stack
    });
    throw error;
  }
};

module.exports = {
  getUsers,
  getUserById,
  createUser,
  updateUser,
  deleteUser
};
```

**src/validators/userValidator.js**
```javascript
/**
 * ユーザーバリデーター
 * @module validators/userValidator
 */

const Joi = require('joi');

/**
 * ユーザー作成・更新のバリデーションスキーマ
 */
const userSchema = Joi.object({
  email: Joi.string().email().required().messages({
    'string.email': 'Invalid email format',
    'any.required': 'Email is required'
  }),
  name: Joi.string().min(2).max(100).required().messages({
    'string.min': 'Name must be at least 2 characters',
    'string.max': 'Name must not exceed 100 characters',
    'any.required': 'Name is required'
  }),
  age: Joi.number().integer().min(0).max(150).optional().messages({
    'number.base': 'Age must be a number',
    'number.min': 'Age must be positive',
    'number.max': 'Age must not exceed 150'
  })
});

/**
 * ユーザーIDのバリデーションスキーマ
 */
const userIdSchema = Joi.object({
  id: Joi.number().integer().positive().required()
});

/**
 * ユーザーデータをバリデート
 */
const validateUser = (req, res, next) => {
  const { error } = userSchema.validate(req.body, { abortEarly: false });

  if (error) {
    const errors = error.details.map(detail => ({
      field: detail.path[0],
      message: detail.message
    }));

    return res.status(400).json({
      error: {
        message: 'Validation failed',
        code: 'VALIDATION_ERROR',
        details: errors
      }
    });
  }

  next();
};

/**
 * ユーザーIDをバリデート
 */
const validateUserId = (req, res, next) => {
  const { error } = userIdSchema.validate(req.params);

  if (error) {
    return res.status(400).json({
      error: {
        message: 'Invalid user ID',
        code: 'INVALID_USER_ID'
      }
    });
  }

  next();
};

module.exports = {
  validateUser,
  validateUserId
};
```

## 実践例2: React コンポーネントの生成

### プロンプト

```
タスク: ユーザープロフィール表示コンポーネントを実装

要件:
- ユーザー情報（名前、メール、プロフィール画像）を表示
- 編集モードとビューモードを切り替え可能
- フォームバリデーション
- ローディング状態の表示
- エラーハンドリング

技術仕様:
- React 18 + TypeScript
- React Hooks (useState, useEffect, useCallback)
- CSS Modules でスタイリング
- プロフィール画像は遅延読み込み

コンポーネント構成:
- UserProfile.tsx - メインコンポーネント
- UserProfile.module.css - スタイル
- UserProfile.test.tsx - テストコード
```

### 生成されるコード

**UserProfile.tsx**
```typescript
import React, { useState, useEffect, useCallback } from 'react';
import styles from './UserProfile.module.css';

interface User {
  id: number;
  name: string;
  email: string;
  avatarUrl?: string;
}

interface UserProfileProps {
  userId: number;
  onUpdate?: (user: User) => void;
}

interface FormErrors {
  name?: string;
  email?: string;
}

/**
 * ユーザープロフィール表示・編集コンポーネント
 */
const UserProfile: React.FC<UserProfileProps> = ({ userId, onUpdate }) => {
  const [user, setUser] = useState<User | null>(null);
  const [isEditing, setIsEditing] = useState(false);
  const [isLoading, setIsLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);
  const [formData, setFormData] = useState<Partial<User>>({});
  const [formErrors, setFormErrors] = useState<FormErrors>({});

  // ユーザーデータを取得
  useEffect(() => {
    const fetchUser = async () => {
      try {
        setIsLoading(true);
        const response = await fetch(`/api/users/${userId}`);
        
        if (!response.ok) {
          throw new Error('Failed to fetch user');
        }
        
        const data = await response.json();
        setUser(data.data);
        setFormData(data.data);
      } catch (err) {
        setError(err instanceof Error ? err.message : 'Unknown error');
      } finally {
        setIsLoading(false);
      }
    };

    fetchUser();
  }, [userId]);

  // フォームバリデーション
  const validateForm = useCallback((): boolean => {
    const errors: FormErrors = {};

    if (!formData.name || formData.name.trim().length < 2) {
      errors.name = '名前は2文字以上で入力してください';
    }

    if (!formData.email || !/^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(formData.email)) {
      errors.email = '有効なメールアドレスを入力してください';
    }

    setFormErrors(errors);
    return Object.keys(errors).length === 0;
  }, [formData]);

  // フォーム送信
  const handleSubmit = async (e: React.FormEvent) => {
    e.preventDefault();

    if (!validateForm()) {
      return;
    }

    try {
      setIsLoading(true);
      const response = await fetch(`/api/users/${userId}`, {
        method: 'PUT',
        headers: {
          'Content-Type': 'application/json',
        },
        body: JSON.stringify(formData),
      });

      if (!response.ok) {
        throw new Error('Failed to update user');
      }

      const data = await response.json();
      setUser(data.data);
      setIsEditing(false);
      
      if (onUpdate) {
        onUpdate(data.data);
      }
    } catch (err) {
      setError(err instanceof Error ? err.message : 'Unknown error');
    } finally {
      setIsLoading(false);
    }
  };

  // 編集キャンセル
  const handleCancel = () => {
    setFormData(user || {});
    setFormErrors({});
    setIsEditing(false);
  };

  // 入力変更ハンドラ
  const handleInputChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const { name, value } = e.target;
    setFormData(prev => ({ ...prev, [name]: value }));
    // エラーをクリア
    if (formErrors[name as keyof FormErrors]) {
      setFormErrors(prev => ({ ...prev, [name]: undefined }));
    }
  };

  if (isLoading) {
    return <div className={styles.loading}>読み込み中...</div>;
  }

  if (error) {
    return <div className={styles.error}>エラー: {error}</div>;
  }

  if (!user) {
    return <div className={styles.error}>ユーザーが見つかりません</div>;
  }

  return (
    <div className={styles.container}>
      <div className={styles.header}>
        {user.avatarUrl && (
          <img
            src={user.avatarUrl}
            alt={`${user.name}のプロフィール画像`}
            className={styles.avatar}
            loading="lazy"
          />
        )}
      </div>

      {isEditing ? (
        <form onSubmit={handleSubmit} className={styles.form}>
          <div className={styles.formGroup}>
            <label htmlFor="name">名前</label>
            <input
              type="text"
              id="name"
              name="name"
              value={formData.name || ''}
              onChange={handleInputChange}
              className={formErrors.name ? styles.inputError : ''}
            />
            {formErrors.name && (
              <span className={styles.errorMessage}>{formErrors.name}</span>
            )}
          </div>

          <div className={styles.formGroup}>
            <label htmlFor="email">メールアドレス</label>
            <input
              type="email"
              id="email"
              name="email"
              value={formData.email || ''}
              onChange={handleInputChange}
              className={formErrors.email ? styles.inputError : ''}
            />
            {formErrors.email && (
              <span className={styles.errorMessage}>{formErrors.email}</span>
            )}
          </div>

          <div className={styles.buttonGroup}>
            <button type="submit" className={styles.submitButton}>
              保存
            </button>
            <button
              type="button"
              onClick={handleCancel}
              className={styles.cancelButton}
            >
              キャンセル
            </button>
          </div>
        </form>
      ) : (
        <div className={styles.view}>
          <div className={styles.field}>
            <span className={styles.label}>名前:</span>
            <span className={styles.value}>{user.name}</span>
          </div>
          <div className={styles.field}>
            <span className={styles.label}>メール:</span>
            <span className={styles.value}>{user.email}</span>
          </div>
          <button
            onClick={() => setIsEditing(true)}
            className={styles.editButton}
          >
            編集
          </button>
        </div>
      )}
    </div>
  );
};

export default UserProfile;
```

## コード品質を高めるテクニック

### 1. 段階的な生成

一度にすべてを生成せず、段階的にアプローチ：

```
ステップ1: 基本構造の生成
  → インターフェース、型定義、骨組み

ステップ2: ビジネスロジックの追加
  → 各機能の実装

ステップ3: エラーハンドリング
  → 異常系の処理

ステップ4: テストコード
  → ユニットテスト
```

### 2. コンテキストの提供

既存コードのパターンを参考例として提示：

```
以下の既存コードと同じパターンで実装してください：

[既存コードのスニペット]

新規実装: ProductController.js
```

### 3. 制約の明示

```
制約:
- ファイルサイズ: 300行以下
- 循環的複雑度: 10以下
- 依存ライブラリ: 既存のもののみ使用
- パフォーマンス: O(n) 以下のアルゴリズム
```

## よくある問題と対策

### 問題1: 生成されたコードが動作しない

**原因**:
- 依存関係の不足
- API の変更
- 環境差異

**対策**:
```
1. 使用するライブラリのバージョンを明示
2. 実行環境を指定（Node.js 18, React 18 など）
3. 段階的に実行して確認
```

### 問題2: 既存コードとの整合性がない

**原因**:
- コーディング規約の未指定
- プロジェクト構造の不一致

**対策**:
```
1. AGENTS.md で規約を明確化
2. 既存コードの例を提示
3. ファイル構造を明示
```

### 問題3: 過度に複雑なコード生成

**原因**:
- 要件が曖昧
- 一度に多くを要求

**対策**:
```
1. 要件を明確化
2. 小さく分割して生成
3. YAGNI原則（You Aren't Gonna Need It）の適用
```
