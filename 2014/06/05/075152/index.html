下書きが多くなりすぎた

2014年6月5日

@nqounetです。

ブックマーク代わりにブログを書こうと思っているのですが、記事が書けるほどわかっていないことが多くなりすぎて溜まってきてます。

このままだと未消化で終わりそうなので、項目を列挙してお茶を濁そうかなと。

使ってみたい


  • YUI Library

  • Skin Builder

ウェブ上でデザインをカスタマイズできるサイトを検索していて発見しました。

Skin Builderがスゴイのですが、設定できることが多くて面倒になってしまって放置しています。

時間がある時に何か作りたいと思うのですが、時間があっても気力がなくて放置したままです。

コレが下書きの一番下にあって、いつも気になっています。

YUIと言えば、yuidocが素敵ですね。

JavaScriptで書いた関数について、メモ程度に書いておくだけでも、コマンド一発でちゃんとしたHTMLに仕上げてくれます。

もちろん、gruntのプラグインもあります。


  • gruntjs/grunt-contrib-yuidoc

正式な使い方はアレですが、ファイルの最初にクラス名、各関数にはメソッド名とパラメータの簡単な説明というように、書く内容が決まっているので、テンプレートを使っているように書けるのが良いです。

普段は面倒であまり書かないのですが、yuidocを使うようになってから、関数ごとにコメントが入ってソースコードも見やすくなってきた気がします。

コレがこのまま納品できれば楽なんですけどねぇ。

フラットアイコンフォント


  • Elusive-Icons Webfont | shoestrap.org

  • aristath/elusive-iconfont

ちゃんと読んでいないのでわかりませんが、たぶんアイコンフォントです。

bootstrapにも、クラスを指定するとアイコンが使えるような仕組みがあります。（Glyfhicons）

また、この手のもので有名なのはFont Awesomeだと思います。


  • Components · Bootstrap

  • Font Awesome, the iconic font and CSS framework

ウェブアプリを作る場合、こういったアイコンがあると便利です。

特にスマートフォンは、解像度の違いがあって、何も考えずに画像を使うとぼやけてしまいます。Retinaディスプレイもそうですね。

なので、画像を使うよりもこういったアイコンのほうが使いやすいと思います。

角丸もCSSでできるようになりましたし、部品としての画像ファイルはほとんど不要な時代になってきたと思います。

デザインが苦手な私としては、こういったツールを上手く使うことで表現に幅を持たせたい、と思っています。
